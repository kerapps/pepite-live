amElnagdy/guard-skills

This is the other half of ponytail, and maybe the better half. guard-skills reviews your agent's output before you do. It looks for the usual ways AI code goes wrong. The confident but broken test. The doc that describes code that isn't there. The flow is simple. Let the agent work. Then run the right guard on the diff before you commit. It's prompt-based, so it won't catch everything a type-checker would. But as a cheap second opinion in Claude Code, Codex, or Cursor, it earns its place fast.

Reward Modeling for Multi-Agent Orchestration

Running many agents? You've hit the eval wall. How do you tell a good run from a lucky one without grading each by hand? OrchRM has an idea. Use the artifacts a multi-agent run already produces. Turn them into win/lose pairs. Train a reward model on those. No human labels. No costly re-runs. It's a preprint and a work in progress, so treat the numbers as a claim, not proof. But it ships code. And the core idea is right: score the orchestration, not the sub-tasks.
